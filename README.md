# Recall

**RAG-powered document search.** Upload your PDFs and text files, then ask questions in plain English and get accurate, AI-generated answers вҖ” each one backed by citations that point to the exact source document and excerpt.

Recall is a full Retrieval-Augmented Generation (RAG) pipeline running on AWS. Documents are stored in S3, their metadata tracked in DynamoDB, their text embedded into vectors in Pinecone, and answers generated by Anthropic's Claude with grounded citations.

> Built by **Omar Faruk** вҖ” University of Connecticut CS, AI/ML concentration.

## Tech Stack

| Layer              | Technology                          | Notes |
|--------------------|-------------------------------------|-------|
| Frontend           | React + Vite                        | Dark UI, drag-and-drop upload, chat + citation cards |
| API Layer          | AWS API Gateway (REST)              | Routes HTTPS requests to Lambdas |
| Compute            | AWS Lambda (Node.js 20.x)           | Four functions: `upload`, `process`, `query`, `documents` |
| Document Storage   | AWS S3                              | Raw uploaded files; versioned, public access blocked |
| Metadata Database  | AWS DynamoDB                        | Per-document status, filename, chunk count, etc. |
| Vector Database    | Pinecone (free tier)               | Stores chunk embeddings for similarity search |
| Embeddings         | Google Gemini (free tier)           | Mocked in Phases 1вҖ“2, real from Phase 3; swappable adapter |
| LLM                | Anthropic Claude API (`claude-haiku-4-5`) | Generates grounded answers + citations (model set via env var) |
| Auth / Security    | IAM (least-privilege per-Lambda roles) | No "god role" вҖ” each Lambda gets only what it needs |
| Version Control    | Git + GitHub                        | `main` branch protected |

## Full deploy (from scratch)

Each phase has a complete, click-by-click guide in [`docs/`](./docs). Run them in order:

1. **[Phase 1 вҖ” AWS setup & storage](./docs/phase1-aws-setup.md):** account + MFA, billing alerts, IAM user, S3, DynamoDB, the `upload` Lambda.
2. **[Phase 2 вҖ” Processing pipeline](./docs/phase2-processing.md):** the `process` Lambda, the S3 trigger, extract вҶ’ chunk вҶ’ embed.
3. **[Phase 3 вҖ” RAG query](./docs/phase3-rag-query.md):** Pinecone + Gemini + Anthropic keys, the `query` Lambda, go live.
4. **[Phase 4 вҖ” React frontend](./docs/phase4-frontend.md):** run the dark UI locally (demo or live).
5. **[Phase 5 вҖ” Polish & deployment](./docs/phase5-polish-deploy.md):** API Gateway + CORS, the `documents` read Lambda, IAM audit, flip the frontend to live.

### Prerequisites
- Node.js 20.x and npm
- AWS account (free tier), GitHub account
- Pinecone free-tier account, Google Gemini API key, Anthropic API key

### Environment variables
- Backend: each Lambda's env vars are documented in [`.env.example`](./.env.example).
- Frontend: see [`frontend/.env.example`](./frontend/.env.example) вҖ” `VITE_USE_MOCK` and `VITE_API_BASE_URL`.
- **Never commit real keys.** `.gitignore` already excludes `.env*` (except the examples).
